
-------------
   General   
-------------

* Use consistent variable and function/method names
* Make test names and code to tell stories 
  instead of introducing explanatory comments.

	* Strive for a test name that summarizes what the test verifies. 
	* Have the test name complete a sentence about the behavior it describes,
	  making it  obvious from its name,
		try to keep it 5-7 words max, when possible.

* Add a concise docstring.


* Introduce meaningful constants when it makes sence.

* Tests can be concise, with a typical range from one to five statements 
  (use fixtures and similar things to achieve this).

* Split larger tests into smaller, more focused tests.
* Move test clutter to helper functions/methods.

* Cover edge cases as well (e.g., invalid inputs, rate limits (when applicable)).



* Use logging.info() instead of print(), unless colorful output is needed.
